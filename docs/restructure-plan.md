## 一、信息架构（IA）

### 1.1 顶层模型

```
┌─────────────────────────────────────────────────────┐
│                    Cognest App                       │
├───────────┬─────────────────────────────────────────┤
│           │                                         │
│  侧边栏   │           主区域                        │
│  (固定)   │     (视图替换 + 返回栈)                  │
│           │                                         │
│  ┌─────┐  │  ┌─────────────────────────────────┐   │
│  │发现  │←─┼──│ AI Feed / 详情视图 / 生成视图    │   │
│  ├─────┤  │  └─────────────────────────────────┘   │
│  │创作  │←─┼──│ 文件列表 + 编辑器 + AI 对话      │   │
│  ├─────┤  │  └─────────────────────────────────┘   │
│  │碎片  │←─┼──│ Inbox 流（时间倒序）             │   │
│  └─────┘  │  └─────────────────────────────────┘   │
│           │                                         │
│  ─ ─ ─ ─ │                                         │
│  Agent 面板│                                         │
│  (状态指示)│                                         │
└───────────┴─────────────────────────────────────────┘

   ＋ 全局快捷键悬浮框（⌘⇧Space → 快速输入碎片）
```

### 1.2 三个对象 + 一个底座

| 类型 | 定义 | 谁负责 | 对应导航 |
|------|------|--------|----------|
| **碎片** | 灵感、摘录、画线（append-only，不可变） | 人输入 | 侧边栏「碎片」 |
| **作品** | 文章、笔记、读后感（从碎片长出来） | 人输出 | 侧边栏「创作」 |
| **视图** | 知识图谱、topic、todolist、回顾（AI 生成） | AI 生成，人浏览 | 侧边栏「发现」内按需浮现 |
| **底座** | 知识库 + MCP server | AI 维护 | 无独立导航（半隐形） |

### 1.3 核心原则

1. **侧边栏只放动词（发现/创作/碎片），不放名词（笔记/图谱/看板）**
2. **名词要么住在动词空间内部，要么是按需生成的视图**
3. **没有"页面跳转"概念——所有深入都是主区域内的视图替换 + 返回栈**
4. **生成式视图是核心差异化——不是固定菜单，是 AI 按需造的看板**

## 三、交互模型

### 3.1 全局导航

- **侧边栏**（固定 248px）：品牌 Logo + 快捷录入按钮 + 3 个导航项 + Agent 状态面板
- **选中态**：白底 + 边框 + 图标变蓝
- **从不折叠**（桌面应用，不做响应式收缩）

### 3.2 发现页交互

```
Feed 卡片流（默认视图）
    │
    ├─ 点击卡片 → 主区域替换为「详情视图」
    │              ├─ 左上角 ← 返回 feed
    │              └─ 内容：完整统计/分析/碎片时间线
    │
    ├─ 生成栏输入 → 主区域替换为「生成视图」
    │              ├─ 左上角 ← 返回 feed
    │              ├─ 右上角 📌 固定（成为 feed 常驻卡片）
    │              └─ 内容：AI 按指令现做的交互式视图
    │
    └─ 关闭卡片（×）→ 卡片动画消失，不再推送
```

### 3.3 创作页交互

```
┌────────────────────────────────────────────────┐
│  侧边栏  │  文件列表(280px)  │  编辑器主区域    │
│          │                    │                 │
│          │  搜索框            │  标题            │
│          │  ─────────         │  ─────────      │
│          │  📄 文章 A  ←active│  Markdown 正文   │
│          │  📄 草稿 B         │                 │
│          │  📄 草稿 C         │  ─────────      │
│          │                    │  AI 建议浮层     │
│          │  [+ 新建]          │                 │
│          │                    │  ─────────      │
│          │                    │  AI 对话面板     │
│          │                    │  (可折叠/展开)   │
└────────────────────────────────────────────────┘
```

- 文件列表：所有作品（草稿+已完成），按最近编辑排序
- 编辑器：Markdown，toolbar 含格式/引用碎片/插入 AI 建议
- AI 对话面板：底部可展开，与 AI 讨论当前文章内容
- 侧边浮现：写作时自动浮出相关碎片（右侧边缘 popover）

### 3.4 碎片页交互

```
Inbox 流（时间倒序）
    │
    ├─ 顶部输入框（与侧边栏快捷录入相同功能，App 内版本）
    ├─ 碎片卡片流
    │   ├─ 每条：内容 + 时间 + AI 标签 + 关联 topic
    │   └─ 长按/右键 → 操作菜单（引用到创作/查看关联/删除）
    └─ AI 已整理的碎片有 ✓ 标记（表示已归入某 topic）
```

### 3.5 全局快捷键悬浮框

- `⌘⇧Space`：从任何位置弹出
- 置于所有窗口之上
- 纯文本输入 + 回车提交
- 提交后自动收起 + toast 确认

## 四、设计系统

### 4.1 视觉方向

Apple 设计系统（已绑定）：
- **配色**：白/浅灰二元节奏，单一蓝强调
- **字体**：SF Pro Display (标题) + SF Pro Text (正文) + SF Mono (数据)
- **风格**：克制留白、边框引导层级、无装饰性渐变

### 4.2 Token 架构（`assets/tokens.css`）

单一文件定义所有设计 token，其他页面通过 `<link>` 引用：

```
Surface:    --bg / --surface / --surface-warm
Foreground: --fg / --fg-2 / --muted / --meta
Border:     --border / --border-soft
Accent:     --accent / --accent-on / --accent-hover / --accent-active
Semantic:   --success / --warn / --danger
Typography: --font-display / --font-body / --font-mono
Scale:      --text-xs ~ --text-2xl
Spacing:    --space-1 ~ --space-12
Radius:     --radius-sm / --radius-md / --radius-lg / --radius-pill
Elevation:  --elev-flat / --elev-ring / --elev-raised
Motion:     --motion-fast / --motion-base / --ease-standard
```

### 4.3 组件规范（`assets/app.css`）

| 组件 | 类名 | 用途 |
|------|------|------|
| App Shell | `.win` | 全屏 flex 容器 |
| 侧边栏 | `.side` | 固定宽度导航 |
| 交通灯 | `.traffic` | macOS 窗口控制 |
| 品牌 | `.brand` | Logo + 产品名 |
| 快捷录入 | `.capture-btn` | 侧边栏录入入口 |
| 导航项 | `.nav a` / `.nav a.on` | 三级导航 |
| Agent 状态 | `.agents` / `.agent` | 底部 Agent 指示器 |
| 卡片 | `.card` | Feed 卡片 |
| 生成栏 | `.gen` | AI 视图生成入口 |
| 徽标 | `.kind` / `.badge` | 分类标签 |
| 主按钮 | `.btn-primary` | 蓝色填充 CTA |
| 次按钮 | `.btn-secondary` | 边框按钮 |
| 输入框 | `.field` | 表单输入 |
| 段落标签 | `.sec-lbl` | Feed 分段标题 |

### 4.4 图标系统

- 全部使用 **1.5px stroke, 18×18, currentColor** 的线性 SVG
- **禁止 emoji 图标**
- 统一从设计系统内联（不引外部图标库）

## 五、内容规范

### 5.1 Feed 卡片类型

| 类型 | 触发条件 | Badge 颜色 |
|------|----------|------------|
| 本周回顾 | 每周一次，Reflection Agent 生成 | 紫色 |
| 成长中的 Topic | 某主题碎片 > 20 条 | 绿色 |
| 新发现关联 | Curator Agent 发现新关联 | 蓝色 |
| 写作建议 | 某 topic 成熟度达阈值 | 橙色 |
| 生成视图（已固定） | 用户手动固定 | 灰色 |

### 5.2 文案原则

- **卡片标题**：动词开头或发现句式（「你本周在…」「Agent 发现…」）
- **数据**：真实可能的范围（碎片 3-50 条/周，topic 5-20 个）
- **不允许**：lorem ipsum、泛用数字（10×、99.9%）、emoji

### 5.3 状态文案

| 场景 | 文案 |
|------|------|
| 空 feed | 「还没有新发现——继续记录碎片，AI 会帮你整理」 |
| 空碎片 | 「按 ⌘⇧Space 记录第一条灵感」 |
| 空创作 | 「从一条碎片开始——点击左上角 + 新建」 |
| 视图生成中 | 「正在为你构建视图…」 |

## 九、风险与取舍

| 决策 | 取舍 | 理由 |
|------|------|------|
| 侧边栏不做响应式 | 放弃移动端适配 | 这是 Mac 桌面应用原型，不是网站 |
| 知识图谱不做固定页 | 用户需要"生成"才能看图谱 | 与「生成式视图」核心主张一致；固定页是杂物抽屉根源 |
| 删除任务看板 | 失去 todo 功能 | 不服务「灵感→认知」主线；如需可通过生成式视图恢复 |
| 共享 CSS 用 `<link>` 引用 | 每个页面不再自包含 | 消灭 token 漂移；原型阶段足够；真实开发会用组件框架 |
| 只做 3 个功能页面 | 不做设置/账户/偏好 | 原型验证 IA 和核心交互，设置页面可后加 |
