# Cognest 产品需求文档 (PRD)

> 版本: 4.0 | 日期: 2026-06-24 | 项目代号: Cognest (Cognition + Nest)

**v4.0 核心变更：**
- 组织原则从「览/作双空间」重构为**「人在两端，AI 在中间」**三面模型
- 页面从 7 个精简至 3 个功能页 + 1 个 launcher
- 设计 token 和公共组件抽为共享 CSS（`tokens.css` + `app.css`），消灭 7 份拷贝
- 生成式视图升为 P0 核心能力（不再是「+ 自定义页面」入口）
- 知识图谱、todolist、Topic 孵化从固定页面变为发现页内的按需生成视图
- 页面模型统一为「主区域视图替换 + 返回栈」，无传统页面跳转

---

## 1. 产品定义

### 1.1 一句话定义

一个帮助用户将碎片化灵感逐渐演化为系统化认知的 **AI 认知工作台**。

### 1.2 组织原则：人在两端，AI 在中间

用户只管两件事：**输入**（记录碎片）和**输出**（创作文章）。中间的整理、发现、归档、关联——全部交给 AI。

这意味着产品的三个面是按「人类动作」切的，不是按「功能分类」或「大脑模式」切的：

| 面 | 人类动作 | AI 角色 | 对应导航 |
|---|---|---|---|
| **碎片** | 记录（输入） | 静默接收，后台聚类 | 侧边栏「碎片」 |
| **创作** | 写作（输出） | 安静辅助，按需响应 | 侧边栏「创作」 |
| **发现** | 浏览（AI 的工作成果） | 主动推送发现 | 侧边栏「发现」（默认首页） |

第四个底座是**知识库 + MCP server**——半隐形，无独立导航，是三个面共享的数据基础设施。

### 1.3 核心设计原则

1. **人只管 I/O，中间交给 AI。** 用户不分类、不打标签、不建文件夹、不维护关联。
2. **侧边栏只放动词，不放名词。** 导航项是「发现/创作/碎片/文章」（你做什么），不是「笔记/图谱/看板」（有什么东西）。
3. **视图是生成式的。** 知识图谱、todolist、某 topic 的全景——都不是固定页面，是 AI 按需生成的。这是产品的核心差异化。
4. **碎片永不删除（immutable log）。** 每条灵感都是潜在价值，即使被合并/总结，原始记录永久保存。
5. **AI 是管家，不是代笔。** AI 可以整理、推荐、发现关联；不替用户判断重要性、不替用户写结论。

## 2. 信息架构

### 2.1 三类对象 + 一个底座

| 类型 | 定义 | 谁负责 | 生命周期 |
|------|------|--------|----------|
| **碎片** | 灵感、摘录、画线 | 人输入 | append-only，不可变 |
| **作品** | 文章、笔记、读后感 | 人输出 | 从碎片长出来，可编辑 |
| **视图** | 知识图谱、topic、todolist、回顾 | AI 生成 | 人浏览，可固定或丢弃 |
| **底座：知识库** | 装着以上全部 + AI 发现的关联 | AI 维护 | MCP 暴露给外部 agent |

关键认知：知识库不是第四类对象，是装着前三类的那张图。视图是它的窗户，MCP 是它的对外接口。

### 2.2 页面结构

```
┌─────────────────────────────────────────────────────┐
│                    Cognest App                       │
├───────────┬─────────────────────────────────────────┤
│           │                                         │
│  侧边栏   │           主区域                        │
│  (固定)   │     (视图替换 + 返回栈)                  │
│           │                                         │
│  ┌─────┐  │                                         │
│  │发现  │←─┼── AI Feed / 详情视图 / 生成视图         │
│  ├─────┤  │                                         │
│  │创作  │←─┼── 灵感/话题/文章面板 + 编辑器 + AI 对话  │
│  ├─────┤  │                                         │
│  │碎片  │←─┼── Inbox 流（时间倒序）                  │
│  ├─────┤  │                                         │
│  │文章  │←─┼── 文章列表 + 预览面板                   │
│  └─────┘  │                                         │
│           │                                         │
└───────────┴─────────────────────────────────────────┘

   ＋ 全局快捷键悬浮框（⌘⇧Space → 快速输入碎片）
```

### 2.3 文件清单

| 文件 | 角色 |
|------|------|
| `index.html` | Launcher（开发用概览 + 各页面入口） |
| `discover.html` | **发现页**（默认首页） |
| `compose.html` | **创作页**（单篇文章工作台） |
| `capture.html` | **碎片页** |
| `articles.html` | **文章页**（全部文章管理 + 预览） |
| `assets/tokens.css` | 设计 token 单一源 |
| `assets/app.css` | 共享组件样式 |

### 2.4 页面模型

**没有「页面跳转」概念。** 所有深入都是主区域内的视图替换 + 返回栈（参考 Linear、Apple Notes、Things）：

- 侧边栏点击 → 切换到对应功能页
- 功能页内点击卡片/项目 → 主区域替换为详情视图（带 ← 返回）
- 生成视图 → 主区域替换为 AI 生成的交互式视图（带 ← 返回 + 📌 固定）

## 3. 核心功能

### 3.1 发现（= 默认首页，AI 主动喂 / Feed）

用户每天打开 Cognest，看到的是 AI 帮他发现了什么——不是从零写字，也不是手动浏览文件。

**Feed 卡片流**：AI 按优先级推送卡片，包括：

| 卡片类型 | 触发条件 | 内容 |
|----------|----------|------|
| 本周回顾 | 每周一次，Reflection Agent | 碎片数/新 topic/活跃方向统计 |
| 成长中的 Topic | 某主题碎片 > 20 条 | topic 名 + 碎片数 + 进度条 + 建议动作 |
| 新发现关联 | Curator Agent 发现跨 topic 关联 | 节点关系图 + 解释 |
| 写作建议 | 某 topic 成熟度达阈值 | AI 分析 + 「开始写作」按钮 |
| 已固定视图 | 用户手动固定 | 缩略展示 + 「展开」 |

**生成栏**：首屏最显眼的输入位——「描述你想看的视图，AI 来现做」。快捷芯片提供常见模板（知识图谱、本月趋势、某 topic 全景、阅读清单）。

**卡片展开**：点击任意卡片 → 主区域替换为详情视图（完整统计/碎片时间线/AI 分析段落），左上角 ← 返回 feed。

**生成视图**：在生成栏输入需求 → AI 生成交互式视图（知识图谱/todolist/趋势图）→ 用户可「固定」到 feed 成为常驻卡片，或一次性查看后返回。

### 3.2 创作（= 单篇文章工作台）

聚焦一篇文章的深度创作空间。从「文章」页点击编辑跳转进入，或直接在创作页新建。

**布局（三栏）**：
- 左栏（280px）：三 tab 切换面板（灵感/话题/文章），内容均为**与当前文章相关**的条目（通过标签、双链、AI 关联匹配）
- 中间主区域：文章标题 + 元信息（状态/字数/最后编辑） + Markdown 编辑器
- 右栏（320px）：AI 对话面板（Writing Agent），讨论当前文章

**左栏三 tab**：
| Tab | 内容 | 关联逻辑 |
|-----|------|----------|
| 灵感 | 与当前文章标签/内容相关的碎片 | 标签匹配 + AI 语义关联 |
| 话题 | 与当前文章关联的 Topic | 共享标签 + 双链 |
| 文章 | 与当前文章关联的其他文章 | 双链 + Topic 共享 |

**核心交互**：
- 点击左栏碎片 → 插入引用到正文（@引用语法 → 生成引用芯片）
- AI 面板主动根据写作上下文推荐碎片和写作建议
- AI 对话面板按需展开，不默认占空间
- 沉浸式模式（`⌘⇧F`）：左栏 + 底部面板全部隐藏，编辑器全屏
- 文件列表中标记状态：草稿 / 已完成

**AI 辅助规则**：
- AI 可推荐结构、扩展思路——但不替用户写结论
- 每条 AI 建议必须有「忽略」按钮
- 沉浸式模式 = 「关掉 AI，让我自己写」的逃生出口

### 3.3 文章（= 全部文章管理）

所有已创建文章的管理页面。从这里预览文章、跳转创作页编辑。

**布局**：
- 左区域：文章列表（搜索 + 状态筛选 + 标签筛选）
- 右面板（380px）：选中文章的预览（标题/状态/字数/标签/内容预览/操作）

**分类与标签系统**：
- AI 自动打标签（基于内容语义分析）
- 用户可手动增删标签
- 标签筛选支持多选组合

**状态管理**：
| 状态 | 含义 | 触发 |
|------|------|------|
| 草稿 | 初次创建、尚未成型 | 新建时默认 |
| 修改中 | 有一定内容但仍在迭代 | 用户手动切换 |
| 已完成 | 定稿，进入知识库 | 用户手动切换 |

**交互**：
- 点击文章行 → 右侧显示预览
- 预览面板内「编辑」按钮 → 跳转到 `compose.html` 打开该文章
- 支持导出 Markdown、删除

### 3.4 碎片（= Inbox 时间流）

用户所有原始输入的落地页。

**布局**：
- 顶部：输入框（与侧边栏快捷录入相同功能，App 内版本）
- 主体：碎片卡片流，按时间倒序
- 每条碎片：内容 + 时间戳 + AI 标签 + 关联 topic 链接

**交互**：
- 回车提交新碎片，零摩擦（不要求分类/标题/标签）
- AI 已整理的碎片有标记（表示已归入某 topic）
- 右键菜单：引用到创作 / 查看关联 / 查看所属 topic
- 支持按时间/标签筛选

### 3.5 快速记录弹窗（⌘⇧Space / 侧边栏按钮）

- App 内任意页面均可触发：快捷键 ⌘⇧Space 或点击侧边栏「快速记录」按钮
- 居中 modal 弹窗，背景模糊遮罩（点击遮罩或按 Esc 关闭）
- 纯文本 textarea 输入 + ⌘↵ 提交
- 提交后自动收起 + 轻量 toast 确认（「已保存碎片」）
- 不要求分类、不弹选项、不打断当前工作
- 未来桌面端支持全局快捷键（App 外也可弹出）

### 3.6 底座：知识库 + MCP

- 碎片、作品、AI 发现的关联全部存入知识库
- AI Agent 持续在知识库上工作（聚类/关联/回顾）
- 通过 MCP server 将知识暴露给外部 agent（如 Cursor、Claude）
- 用户不直接操作知识库——它是隐形的基础设施
- 后期可通过插件生态接入外部数据源（微信读书 skill 等）

## 4. AI Agent 体系

### 4.1 四个 Agent

| Agent | 职责 | 触发方式 | 可见性 |
|-------|------|----------|--------|
| **Curator** | 聚类、标签、关联发现 | 碎片新增时自动触发 | 不可见，成果通过 Feed 推送 |
| **Writing** | 写作辅助、提纲、扩展 | 在创作页被显式激活 | 可见（对话面板 + 建议浮层） |
| **Librarian** | 知识库维护、重复检测、关系维护 | 持续后台运行 | 不可见，成果是图谱/关联 |
| **Reflection** | 每日/周/月回顾 | 周期触发 | 通过 Feed「本周回顾」卡片可见 |

### 4.2 Agent 设计约束

- 每个 Agent 的主动行为都必须有用户可一键关闭或跳过的出口
- Agent 不发通知、不弹窗、不打断用户
- Agent 的工作成果统一通过「发现」feed 呈现，不散落在各处
- 侧边栏不再有 Agent 面板，改为底部状态栏显示当前活跃 Agent

## 5. 交互规范

### 5.1 导航系统

| 组件 | 规格 |
|------|------|
| 侧边栏宽度 | 展开态 248px，收起态 60px（仅显示图标） |
| 导航项 | 4 个：发现/创作/碎片/文章 |
| 选中态 | 白底 + `var(--elev-ring)` + 图标变 `var(--accent)` |
| 快捷录入按钮 | 侧边栏顶部，边框按钮，点击弹出快速记录 modal |
| 侧边栏收起/展开 | 底部 toggle 按钮，收起后只保留图标导航 |
| 底部状态栏 | 固定在窗口底部 28px，显示：当前 Agent 状态、后台任务、知识库条目数、同步状态、设置入口 |

### 5.2 视图替换模型

所有页面内深入遵循统一的视图替换逻辑：

```
顶级视图（feed/文件列表/inbox）
    ↓ 点击
替换为详情视图
    ├── 左上角「← 返回」按钮
    ├── 动画：新视图从右侧滑入 / 旧视图淡出
    └── 浏览器 history.pushState 管理（支持浏览器后退）
```

### 5.3 卡片交互

| 操作 | 行为 |
|------|------|
| hover | `box-shadow: var(--elev-raised)` |
| 点击 | 展开为详情视图（视图替换） |
| × 按钮 | 动画消失（淡出 + 高度收缩），不再推送 |
| 「固定」 | 生成视图变为 feed 常驻卡片 |

### 5.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| `⌘⇧Space` | 快速记录弹窗（全局，App 内任何页面） |
| `⌘⇧F` | 创作页沉浸式模式 |
| `⌘N` | 新建作品 |
| `⌘/` | 发现页聚焦生成栏 |
| `⌘\` | 侧边栏收起/展开 |
| `⌘↵` | 快速记录弹窗内提交保存 |
| `Esc` | 关闭弹窗 / 返回上一级视图 |

### 5.5 快速记录弹窗

- 点击侧边栏「快速记录」按钮 或 按 ⌘⇧Space 触发
- 居中 modal，背景模糊遮罩
- 纯文本 textarea，placeholder「记下你的想法…」
- ⌘↵ 保存，Esc 关闭
- 保存后 toast 确认「已保存碎片」，自动收起

### 5.6 设置面板

- 底部状态栏右侧「设置」入口 或 快捷键触发
- macOS 偏好设置风格 modal（左侧 tab 导航 + 右侧内容区）
- 5 个 tab：
  - **账户**：用户名、邮箱、订阅方案
  - **AI 模型**：默认模型选择、各 Agent 模型配置、API Key
  - **快捷键**：全局快捷键列表（只读展示，后期支持自定义）
  - **知识库**：存储位置、碎片总数、MCP 服务状态
  - **插件**：已安装插件列表（当前为空，预留插件生态入口）

### 5.7 底部状态栏

- 固定窗口底部，高 28px，IDE 风格信息条
- 左侧：当前活跃 Agent + 后台任务描述
- 右侧：知识库条目数、同步状态、设置入口
- Agent 状态取代了原侧边栏底部的 Agent 面板（简化为一行信息）

## 7. 设计规范

### 7.1 视觉方向

Apple 设计系统 — 克制、实用、舒适。

### 7.2 Token 架构

所有设计 token 定义在 `assets/tokens.css` 单一文件中，各页面通过 `<link rel="stylesheet" href="assets/tokens.css">` 引用。**禁止在页面内重复定义 `:root` token。**

完整 token 清单参见 `assets/tokens.css` 文件。核心色板：

| 用途 | Token | 值 |
|------|-------|-----|
| 主背景 | `--bg` | `#ffffff` |
| 面板/侧边栏 | `--surface` | `#f5f5f7` |
| 正文 | `--fg` | `#1d1d1f` |
| 辅助文字 | `--muted` | `#6e6e73` |
| 边框 | `--border` | `#d2d2d7` |
| 主操作 | `--accent` | `#0071e3` |

### 7.3 图标系统

- 全部使用 1.5px stroke, 18×18, `currentColor` 线性 SVG
- **禁止 emoji 图标**
- 统一内联，不引外部图标库

### 7.4 关键设计约束

- 蓝色 accent 最多 2 处可见使用/屏
- 无紫色渐变、无 emoji 图标、无左侧色条卡片
- 圆角分层：控件 8px / 卡片 12-18px / 胶囊 980px
- 阴影克制：`var(--elev-raised)` 为最强层级
- 动画曲线统一：`var(--ease-standard)`

## 8. 技术方向

> 详细技术方案见 `tech-architecture.md`。本节为摘要。

### 8.1 架构选择

| 层面 | 选择 | 原因 |
|------|------|------|
| 桌面框架 | **Tauri v2** | 系统 WebView，Mac 原生体验，包体小，Tolaria 已验证 |
| 前端 | **React + TypeScript** | 复杂交互，类型安全 |
| 构建 | **Vite** | 快速 HMR，Tauri 官方推荐 |
| 编辑器 | **TipTap / ProseMirror** | 富文本 + Markdown 双模式 |
| 数据事实来源 | **纯文本文件（.md + YAML frontmatter）** | git 友好、人可读、immutable log |
| 本地索引 | **SQLite + FTS5** | 可丢弃的派生索引，全文检索 |
| 向量 | **不用向量数据库** | embedding 当 BLOB 存 SQLite，单用户规模暴力余弦够用 |
| AI Embedding | **本地**（fastembed-rs / bge-small-zh） | 高频、隐私、零成本 |
| AI Generation | **云端 API + 可选 Ollama** | 低频、重质量，用户配 key |
| 同步 | **git 同步到 GitHub** | 面向程序员/agent；未来加云端同步服务 |
| Agent 实现 | **prompt + 受限工具 + 结构化输出** | 不用 agent 框架，不接外部 coding agent |

**不用的技术：** 向量数据库（LanceDB）、CRDT（Yjs）、Docker、LangChain/CrewAI、ACP/外部 coding agent。

### 8.2 数据层核心设计

- **文件是唯一事实来源**：每条碎片 = `capture/yyyy/mm/<uuid>.md`，文章 = `articles/*.md`
- **SQLite 是可丢弃索引**：坏了删掉从文件重建
- **git commit history = immutable log**：碎片文件一旦创建正文永不修改
- **AI 派生数据与原文分离**：标签/关联写到 frontmatter 或 `.meta/`，不改正文

### 8.3 AI Agent 体系

四个 Agent 是后台 Job，不是常驻 agent：

| Agent | 触发 | 核心动作 |
|-------|------|----------|
| Curator | 新碎片入库 | 向量聚类（本地免费）+ LLM 命名 |
| Writing | 用户显式激活 | 读上下文 → 对话式辅助 |
| Librarian | 每周定时 | 一致性扫描 + 关系维护 |
| Reflection | 每日/周/月 | 生成回顾 Feed 卡片 |

### 8.4 MCP Server

方向：**暴露知识库给外部 agent**（Cursor/Claude 查你的知识库），而非接入外部 agent 执行。

- 工具：`search_fragments`、`get_topics`、`get_relations`、`get_article`
- 安全：显式开关 + 默认只读 + 不暴露原始全文除非授权
- 自动生成 `AGENTS.md` 告诉外部 agent 你的约定

### 8.5 插件生态（预留）

- 插件 = adapter，把外部数据转成 `capture/*.md` 追加进仓库
- 碎片来源插件：flomo 导出、Apple 备忘录（快捷指令）、微信读书、RSS
- 视图模板插件：自定义的 view spec type
- 当前不实现，架构上预留扩展点

### 8.6 平台策略

| Phase | 平台 | 策略 |
|-------|------|------|
| 0-2 | **macOS 桌面端** | Tauri v2，Mac 原生体验优先 |
| 4 | **Windows 桌面端** | Tauri 换 build target，基本免费 |
| 5 | **移动端** | 不跑核心逻辑，只做数据导入插件（备忘录/flomo → capture/） |

## 10. 使用场景

### 场景 1 — 碎片捕捉（全天，每次 5-15 秒）

Yue 按 `⌘⇧Space`。悬浮输入框出现。打一行字，回车。弹窗消失。全程未打开 Cognest 主界面。后台 Curator Agent 已将这条碎片与相关 topic 关联。

### 场景 2 — 发现回顾（打开 App，5-10 分钟）

Yue 打开 Cognest，进入发现页。AI 推送了本周回顾卡片：「过去 7 天记录 28 条碎片，Agent 架构方向持续活跃」。她点开卡片看详情，又在生成栏输入「给我看 Agent 相关的知识图谱」。AI 现场生成一个交互式图谱，她浏览了两分钟，固定到 feed。

### 场景 3 — 深度写作（2-3 小时）

Yue 进入创作页，当前文章是一篇草稿。左侧面板自动切换到「灵感」tab，展示与该文章相关的碎片。她写到卡住时，在右侧 AI 对话面板问：「这里需要一个反馈回路案例」。AI 推荐引用一条旧碎片。她点击左侧碎片插入引用后继续写。完成后按 `⌘⇧F` 进入沉浸式模式做最终修改。

### 场景 4 — 知识探索（偶尔，20 分钟）

Yue 在发现页生成栏输入「我最近三个月的知识演化图谱」。AI 生成一个节点图，她发现 RAG 和 Context Engineering 之间出现了新的强关联。这个关联是 Librarian Agent 自动维护的。
